# EA Tool Documentation

Welcome to the EA Tool documentation. This tool provides an API-first approach to enterprise architecture management with ArchiMate 3.2 alignment.

## Documentation Structure

- [System Overview](./system-overview.md) - Architecture, design principles, and technology stack
- [Entity Guide](./entity-guide.md) - Detailed descriptions of all domain entities and their usage
- [Relationship Modeling](./relationship-modeling.md) - How to model relationships between entities
- [ArchiMate Alignment](./archimate-alignment.md) - How the tool maps to ArchiMate 3.2 concepts
- [API Usage Guide](./api-usage-guide.md) - How to interact with the API
- [Authorization Guide](./authorization-guide.md) - Authentication and authorization patterns

## Quick Start

1. Review the [System Overview](./system-overview.md) to understand the architecture
2. Read the [Entity Guide](./entity-guide.md) to understand the domain model
3. Check the [Relationship Modeling](./relationship-modeling.md) guide to learn how to connect entities
4. Refer to the [API Usage Guide](./api-usage-guide.md) for integration patterns

## API Documentation

The OpenAPI specification is available at:
- Development: `http://localhost:8000/openapi.json`
- Staging: `https://staging.api.example.com/openapi.json`
- Production: `https://api.example.com/openapi.json`

Interactive API documentation (Swagger UI):
- Development: `http://localhost:8000/docs`
- Staging: `https://staging.api.example.com/docs`
- Production: `https://api.example.com/docs`
